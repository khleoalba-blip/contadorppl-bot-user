# 2026-06-17 — Proyecto ContadorPPL Dashboard

## Proyecto principal
El usuario tiene un bot WhatsApp Baileys llamado "ContadorPPL" para gestión de lotería (Florida, Georgia, New York). Se creó una app Flutter dashboard para Android + API REST integrada al bot.

## Entregables
- **API**: `lib/api_dashboard.cjs` — 15 endpoints REST con auth JWT + código WhatsApp. Integrada al `index.js` del bot.
- **App Flutter**: `flutter_dashboard/` — 37 archivos, Android, Material 3, Provider state management.
- **CI/CD**: `.github/workflows/build.yml` — GitHub Actions, build 3 APKs por ABI, release automático.
- **Repo GitHub**: `khleoalba-blip/contadorppl-bot-user` (rama `main`)

## Bugs encontrados y corregidos (múltiples iteraciones)
1. **Android v1 embedding**: Flutter 3.24+ rechaza embedding v1. Se migraron 5 archivos Gradle/Android a v2 (settings.gradle, build.gradle, app/build.gradle, styles.xml, AndroidManifest.xml).
2. **CardThemeData → CardTheme**: API de Flutter 3.24 cambió el nombre.
3. **StatusBadge imports faltantes**: `jornada_card.dart` y `jornada_detail_screen.dart`.
4. **ic_launcher inexistente**: Quitada referencia del manifest.
5. **APKs se sobreescribían**: Flutter produce `app-release.apk` para cada ABI, pisando el anterior. Fix: copiar inmediatamente después de cada build.
6. **Login no avanzaba (bug arquitectónico)**: `requestCode()` cambiaba `authProvider.status` a `loading`, lo que causaba que el `Consumer` en `main.dart` destruyera el `LoginScreen` y mostrara `SplashScreen`. El `async _solicitarCodigo()` corría en widget disposed. Fix: no cambiar status a loading en `requestCode`, manejar loading localmente.
7. **Loterías incorrectas**: Hardcodeado Charada/Dominical/Quiniela. Cambiado a Florida/Georgia/New York.
8. **Premios no se guardaban**: `api_service.updateGroupConfig()` envolvía body en `{config: ...}`, API espera campos al nivel raíz.
9. **Listeros no se agregaban**: Flutter enviaba `name`, API espera `nombre`.
10. **Jornadas no cargaban**: API buscaba en `data/groups/<id>/*.json` pero las jornadas están en `data/groups/<id>/jornadas/*.json`. Fix: leer de subdirectorio `jornadas/` + MongoDB cache. Se agregó `obtenerTodasJornadasGrupo()` a `db.cjs`.

## Servidor del bot
- URL Render: `https://cppl-stble-2.onrender.com/`
- Puerto: 3000 (interno)
- `api_dashboard.cjs` necesita los fixes de jornadas para buscar en subdirectorio `jornadas/`

## Lecciones técnicas
- Flutter: Nunca cambiar `ChangeNotifier.status` que controla qué pantalla muestra un `Consumer`, desde una operación iniciada dentro de esa pantalla (el widget se destruye y el async callback muere).
- Flutter CI: `--target-platform` produce siempre `app-release.apk`, pisándose. Copiar tras cada build.
- GitHub Actions: `working-directory` a nivel `defaults.run` cuando el proyecto no está en raíz del repo.
- Baileys bot: Jornadas se guardan en `data/groups/<id>/jornadas/<jornadaId>.json`, no en el directorio del grupo directamente.
- Render free tier: timeout mínimo 60s por cold start.
